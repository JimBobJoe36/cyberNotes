----------------------------
Layer 5 - Session Layer
----------------------------
What does it do?
    > Establishes, maintains, and terminates connections between applications
    > Layer 5 = "Are we still talking?"
    > Examples:
        > Staying logged in
        > A persistent SSH connection
        > Cookies/session IDs/tokens (help track you)


----------------------------
Layer 6 - Presentation Layer
----------------------------
What does it do?
    > "Data Massaging"
    > Layer 6 = "How the message is packaged"
        > Encryption (SSL/TLS)
        > Encoding/fornat (JSON/HTML)
        > Compression (JPEG, GIF, MPEG)
    > Ex: HTTPS encrypts web traffic before transmission.


----------------------------
Layer 7 - Application Layer
----------------------------
What does it do?
    > Direct user interaction with applications
    > Provides network services to applications
    > Example Protocols
        > HTTP/HTTPS (Web browsing)
        > FTP (File Transfer Protocol)
        > SMTP (Email)
        > DNS (Domain Name System)

----------------------------
Intro to DNS (Domain Name System): Port 53
----------------------------
What does it do?
    > DNS converts human-readable domain names (google.com) to IP addresses
    > Essential for internet navigation
    > Foward DNS:
        > Resolves a domain name to an IP
    > Reverse DNS (rDNS): 
        > Resolves an IP to a domain name
        > Ex.
            > 142.250.63.110 -> google.com
    
    > PowerShell command: nslookup [website]
        > Used to find the IP address of websites in Windows Command line

----------------------------
Dynamic Host Configuration Protocol (DHCP)
----------------------------
Before DHCP: Static IP Assignment
    > Manually assigning IP addresses to each device is slow/error-prone
    > IP conflicts occur if two devices get the same IP

What is DHCP?
    > Autoassigns IP addresses
    > Runs on a DHCP server that assigns IPs dynamically
    > DHCP assigns...
        > IP Address
        > Subnet Mask
        > Default Gateway
        > DNS Address

OSI Tie-in:
    > DHCP is a Layer 7 Protocol
    > It broadcasts on the local network
    > It relies on lower layers to deliver messages
        > Uses UDP port 67
        > Broadcast is used early because the client doesn't know the server yet

How DHCP Works:
    > The four-step DORA process
        1. Discover - Client broadcasts "any DHCP server?"
        2. Offer - Server offers an IP address
        3. Request - Client requests to use the offered IP
        4. Acknowledge - DHCP server approves and leases the IP

What is inside a DHCP Server?
    > IP address pool (range of available IPs)
    > Lease time (How long an IP keeps an IP)
    > Subnet Mask, Gateway, DNS settings
    > Reservation options for static assignments (e.g., printers, servers)

Why is DHCP a security concern?
    > Rogue DHCP Servers - Malicious or misconfigured servers that hand out bad IP settings
        > Man-in-the-Middle Attacks - Fake DHCP servers can redirect traffics
    > Denial of Service (DoS) - attackers can exhaust the DHCP pool, preventing devices from getting IPs

How to protect DHCP from bad actors?
    > DHCP Snooping
        > Network switch labels ports
            > Trusted: Real DHCP server on this port
            > Untrusted: Blocks DHCP offers/acknowledgements

HTTP Requests/Back End Development
----------------------------

Frontend vs. Backend
    > Frontend: The part of the website the users interact with (HTML, CSS, JavaScript)
    > Backend: The server-side logic that processes requests and manages data (PHP, Python, etc.)
    > HTTP connects the frontend and the backend 
    > API endpoints are backend 'doors' your frontend can knock on (like ports)

What is HTTP?
    > HyperText Transfer Protocol
        > Port 80
        > OSI Layer 7
    > Used for communication between web browsers and servers
    > Stateless, plainext protocol
    > Nowadays, clients are stateless, whereas servers are at rest.

HTTP Request and Response:
    > Request: Sent by the client to the server
        > Method
        > URL
        > Headers
    > Response: Sent by server back to client
        > Status Code
        > Headers
        > Data (HTML, JSON, etc.)

Common HTTP/HTTPS Methods:
    > Get -> Read (Retrieves data, such as a webpage)
    > Post -> Create (Send data; form submission)
    > Put -> Update (Updates a resource)
    > Delete -> Remove (Removes a resource)

Common Status Codes:
    > 100 (Informational)
    > 200 (Success)
    > 300 (Redirection)
    > 400 (Client Errors)
        > 400 (Bad Request)
        > 401 (Unauthorized)
        > 403 (Forbidden)
        > 404 (Not found)
    > 500 (Server Errors)
        > 500 (Internal Server Error)
        > 502 (Invalid Response)

JSON (JavaScript Object Notation) - Lightweight data format

Curl
    > Command-line tool to make HTTP requests
    > Used to retrieve or send data from URLs
    > Basic Commands:
        > Fetch a webpage
            > curl [WEBPAGE]
        > View headers 
            > curl -l [WEBPAGE]
        > Fetch JSON data:
            > curl [WEBPAGE]
        > Make a POST request
            > curl -X POST -d "name=John..."

Postman
    > Graphical tool for testing APIs
    > Allows sending GET, POST, PUT, DELETE requests
    > Easier to visualize requests and response data

----------------------------
Remote Access Solutions (SSH [Secure Shell], RDP [Remote Desktop Protocol], VNC [Virtual Network Computing])
----------------------------
What is Remote Access?
    > Remote access allows users to connect and interact with devices over a network 
    > Used by network admins, hackers, and IT professionals

What is SSH?
    > SSH is an encrypted (AES) remote access protocol
        > Controlled trough console (No GUI)
    > Port 22
    > Found at layers 5,6,7
        > 5: Manages SSH sessions
        > 6: Encrypts transmitted data
        > 7: Provides command line remote access

What is RDP?
    > Remote control of desktops with a GUI 
    > Developed by Microsoft for Windows-based connections
    > New session for each user connected 

What is VNC:
    > VNC is a cross-platform RDP 
        > Supports Windows, Mac, Linux
    > Primarily GUI, reflecting the host machine
    > Mirrors host screen
        > Multiple users will see/control ONE session 

The Firewall Problem
    > Companies block ports (for private reasons)
    > Remote access can bypass firewalls as a backdoor

SSH Tunneling
    > Access restricted ports/services on a device by using port 22 to get access
